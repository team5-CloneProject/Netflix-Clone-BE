## Check List
- 제목 첫 글자는 대문자로 작성했나요?
- 제목은 명령문으로 작성했나요?
- 제목 끝에 마침표(.) 금지
- 제목과 본문을 한 줄 띄워 분리하기
- 본문에 여러줄의 메시지를 작성할 땐 "-"로 구분했나요?


## commit type
- feat     : 기능 (새로운 기능)
- fix      : 버그 (버그 수정)
- update     : 기능 보완 (업그레이드)
- refactor : 리팩토링
- style    : 스타일 (코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)
- docs     : 문서 (문서 추가, 수정, 삭제)
- test     : 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
- chore    : 기타 변경사항 (빌드 스크립트 수정 등)
